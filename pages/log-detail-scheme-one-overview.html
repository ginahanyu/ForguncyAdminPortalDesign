Scheme One: Per-Item Log Detail

Scheme One allows each server command and each scheduled task to define its own log detail level directly in the item details panel. This gives administrators a simple way to control logging precision at the item level without leaving the current tab.

Feature Summary

In Scheme One, the log detail setting is part of the details area on the right side of the Server Commands and Scheduled Tasks tabs.

  • Each server command can have its own log detail configuration.
  • Each scheduled task can have its own log detail configuration.
  • The setting appears at the end of the details panel for the selected item.
  • The configuration is easy to review because it is shown together with the rest of the item information.

Available Options

The log detail selector provides the following options:

Inherit Global Use the same log detail level configured in Advanced > The detail of the server command log.
No Log Do not record detailed log content for the selected item.
Simplify Record a lightweight log with the most important execution information.
Normal Record a balanced amount of detail for routine troubleshooting and review.
Detailed Record the most complete execution details for deeper investigation.

Where It Appears

Server Commands

When a server command is selected from the list on the left, its details are shown on the right. The Log Detail field appears at the bottom of that details area.

Scheduled Tasks

When a scheduled task is selected from the table on the left, its details are shown on the right. The Log Detail field also appears at the bottom of the task details area.

Help Information

A help indicator is displayed next to the Log Detail title. Hovering over it shows the following explanation:

Inherit Global uses the same log detail setting configured for “The detail of the server command log” in Advanced settings.

Why This Scheme Is Useful

  • Clear ownership: the setting belongs to the selected item and is easy to understand in context.
  • Fast adjustment: administrators can change logging without opening an extra configuration dialog.
  • Flexible control: high-risk items can use more detailed logs while common items stay lightweight.
  • Consistency: the same interaction pattern is used for both server commands and scheduled tasks.
